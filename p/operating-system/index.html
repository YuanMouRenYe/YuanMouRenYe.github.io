第一章 引论

操作系统的主要功能

功能 内容
处理机管理 进程/线程调度、分配 CPU
存储器管理 内存分配、虚拟存储、地址映射
设备管理 I/O 分配、缓冲、驱动
文件管理 文件存取、目录、权限
用户接口 命令行、图形界面等（原文「窗口管理」多指用户接口层）

操作系统在计算机系统中处于硬件与用户/应用程序之间，是对硬件资源的抽象与管理者。

操作系统的特征

特征 含义
并发 两个或多个事件在同一时间间隔内发生（宏观同时、微观交替）
共享 资源可供多个并发进程共同使用（互斥共享 / 同时访问）
虚拟 把一个物理实体变为多个逻辑对应物（虚拟 CPU、虚拟内存等）
异步 进程以不可预知的速度推进，需同步机制协调

操作系统的基本类型

类型 特点
批处理 多道程序并发，提高资源利用率，少交互
分时 时间片轮转，用户可交互，响应及时
实时 优先处理紧急事件，强调实时性与可靠性

第二章 进程的描述与控制

进程的概念

  • 定义：程序的一次动态执行过程（程序是静态的）。
  • 组成：PCB（进程控制块）+ 程序段（代码）+ 数据段（运行数据）。

进程的特征

动态性、并发性、独立性、异步性、结构性（由 PCB 描述）。

三级调度

调度 作用
高级（作业）调度 从外存后备队列选作业进入内存，建立进程，送入就绪队列
中级调度 挂起/唤醒：内存紧张时把暂不运行进程换出；宽松时再换入
低级（进程）调度 从就绪队列选进程，分配 CPU，进入运行态

进程状态转换

转换 典型原因
运行 → 就绪 时间片用完；被更高优先级进程抢占
运行 → 阻塞 等待 I/O、等待资源、等待某事件发生
阻塞 → 就绪 等待的事件发生（I/O 完成等）
就绪 → 运行 被调度程序选中

进程通信（IPC）

方式 说明
共享存储 映射同一块内存区域
消息传递 消息头 + 消息体，send/receive
管道 半双工/全双工字节流（匿名管道、命名管道）

线程

对比 进程 线程
定位 资源分配基本单位 调度/执行基本单位
并发 进程间可并发 进程内线程也可并发
控制块 PCB TCB（含线程 ID）
状态 就绪、运行、阻塞 同样三种基本状态

实现级别

类型 说明
用户级线程 线程库在用户空间调度，内核无感知
内核级线程 内核参与调度；常见 1:1（一用户线程对应一内核线程）

第三章 处理机调度与死锁

调度方式

方式 说明
非抢占 进程占用 CPU 直到结束或主动放弃
抢占 有更重要/更紧迫进程时，剥夺当前 CPU

调度评价指标

指标 公式 / 含义
周转时间 完成时间 − 到达时间
带权周转时间 周转时间 ÷ 服务时间
等待时间 周转时间 − 服务时间（或 在就绪队列中等待之和）
响应时间 提交到首次响应（分时系统重要）
响应比 （等待时间 + 服务时间）÷ 服务时间
CPU 利用率 CPU 忙碌时间 ÷ 总时间
系统吞吐量 完成作业数 ÷ 总时间

调度算法

1. 先来先服务（FCFS）

按到达顺序，非抢占。实现简单，短作业等待长作业时平均周转时间可能较大。

2. 短作业优先（SJF）

优先服务时间短的进程。

类型 规则
非抢占 SJF 当前进程运行完，在已到达进程中选服务时间最短
抢占 SJF（SRTN） 新进程到达时，若其服务时间更短则抢占

3. 优先级调度

按优先级高者优先；同优先级可按 FCFS。分抢占 / 非抢占。

4. 高响应比优先（HRRN）

非抢占，每次选择响应比最高的进程。

5. 时间片轮转（RR）

就绪队列 FCFS，每进程运行一个时间片，用完排到队尾。

优点 缺点
公平、响应快，适合分时 时间片难选；不区分紧急程度；上下文切换开销

6. 多级反馈队列（MLFQ）

  • 多级队列，优先级从高到低，时间片从小到大
  • 新进程进入最高级队列，FCFS + RR
  • 时间片用完未结束 → 降到下一级队尾
  • 已在最低级则仍在最低级队尾轮转

死锁

各进程互相等待对方持有的资源，导致都无法推进。

产生必要条件（四者同时成立）

  1. 互斥：资源只能互斥使用
  2. 占有并等待：持有资源的同时等待其他资源
  3. 不可剥夺：资源只能由持有者主动释放
  4. 循环等待：存在进程资源的环形等待链

处理策略

策略 思路
预防 破坏四条件之一（如按序分配资源）
避免 分配前判断是否进入不安全状态（银行家算法）
检测 允许发生，定期检测并恢复
解除 剥夺资源、回滚、杀死进程等

安全状态 → 一定不会死锁；不安全状态 → 可能死锁。

银行家算法（避免死锁）

矩阵 含义
Available 可用资源向量
Max 各进程最大需求
Allocation 已分配
Need 还需资源，Need = Max − Allocation

安全序列：若存在一种分配顺序使所有进程都能完成，则当前为安全状态。

死锁检测（资源分配图）

  • 方框：资源类；圆：进程
  • 资源 → 进程：已分配；进程 → 资源：正在请求
  • 资源耗尽 + 循环等待 是死锁关键特征

第四章 进程同步

基本概念

概念 说明
进程同步 多进程按一定次序协作，保证可再现性
进程互斥 访问临界资源时排他，同一时刻仅一进程进入临界区
临界资源 一次仅允许一个进程使用的资源（打印机、共享变量等）

信号量机制

原语 别名 含义
wait(S) P 申请资源，S--；若 S<0 则阻塞
signal(S) V 释放资源，S++；若 S≤0 则唤醒等待者
用途 初值 用法
互斥 S = 1 进入临界区 P，离开 V
同步 S = 0 前驱操作后 V，后继操作前 P

经典问题

生产者—消费者

角色 操作语义
生产者 P(空缓冲区)；生产；V(满缓冲区/产品)
消费者 P(产品)；消费；V(空缓冲区)

常用信号量：mutex=1（互斥访问缓冲区）、empty=n、full=0。

读者—写者

  • 多个读者可同时读
  • 写者互斥，且读写互斥
  • 用 readcount + mutex 保护计数；写者用 rw_mutex

第五章 存储器管理

目标

解决「程序逻辑地址空间 > 物理内存」：覆盖、交换、分页、分段、段页式。

连续分配

1. 单一连续分配

系统区（低地址）+ 用户区（用户进程）。简单，无碎片，利用率低。

2. 固定分区

内存划分为固定大小分区，每区一道作业。有内部碎片。

3. 动态分区

按进程需要动态划分，有外部碎片，需紧凑。

动态分区分配算法

算法 策略
首次适应 FF 从头找第一个足够大的空闲区
最佳适应 BF 选最小且够用的空闲区（易留小碎片）
最坏适应 WF 选最大空闲区
邻近适应 NF 从上次查找结束位置继续（首次适应的改进）

分页存储管理

逻辑地址 → 页号 + 页内偏移；物理地址 → 块号 + 块内偏移。

1
2
3

页号   = 逻辑地址 / 页面长度（整除）
页内偏移 = 逻辑地址 % 页面长度
物理地址 = 页表[页号].块号 × 页大小 + 页内偏移

概念 说明
页表 页号 → 块号映射
TLB 快表，加速地址变换
多级页表 节省页表连续空间

第六章 虚拟存储器

在分页基础上，只把当前需要的页调入内存，其余在辅存；缺页时页面置换。

页面置换算法

算法 规则 特点
OPT 淘汰未来最久不再访问的页 理论最优，无法实现
FIFO 淘汰最先进入内存的页 简单，可能 Belady 异常
LRU 淘汰最近最久未使用的页 性能好，实现成本高
Clock 环形队列 + 访问位 LRU 近似，常用

原文「最近醉酒」为 LRU（Least Recently Used） 之笔误。
OPT 中「置换次数」= 发生缺页且需换出的次数（内存已满后的换页）。

工作集与抖动

  • 抖动（颠簸）：频繁缺页，换页时间 > 执行时间
  • 缓解：增加物理块、局部置换、工作集算法

第七章 输入输出系统

磁盘调度（减少寻道时间）

假设当前磁头位置已知，请求队列给定。

算法 规则
FCFS 按请求到达顺序服务
SSTF 每次选距磁头最近的柱面（可能饥饿）
SCAN（电梯） 单向扫描到端点再反向；端点内只朝一个方向服务
C-SCAN 循环扫描，只朝一个方向处理请求，到端点快速返回起点

原文「按照要地址的顺序」多指 FCFS；「莫个方向」为 某个方向（C-SCAN）。

I/O 控制方式（补充）

方式 说明
程序直接控制 CPU 轮询
中断驱动 I/O 完成中断通知
DMA 块设备与内存直接传，少占 CPU
通道 专用 I/O 处理器
