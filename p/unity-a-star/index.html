字段 含义

hCost 从当前节点到终点的启发式估计代价
FCost gCost + hCost，A* 排序依据
parent 最短路径上的前驱节点

public void CreateMap()
{
    Map = new Node[mapX, mapY];

    for (int x = 0; x < mapX; x++)
    {
        for (int y = 0; y < mapY; y++)
        {
            // 确定地图原点（第一象限）
            Vector2Int pos = new Vector2Int(mapOrigin.x + x, mapOrigin.y + y);
            GameObject obj = Instantiate(nodePrefab, new Vector3(pos.x, pos.y, 0), Quaternion.identity, mapRoot);

            Node node = obj.GetComponent<Node>();
            node.gridPos = new Vector2Int(x, y); // 世界坐标 -> 网格虚拟坐标
            node.isWalkable = true;

            Map[x, y] = node;
        }
    }
}

注：图中黑色为真实世界坐标，红色为虚拟网格坐标（参与路径计算）。

public List<Node> GetNeighbors(Node node)
{
    List<Node> neighbors = new List<Node>();
    for (int x = -1; x <= 1; x++)
    {
        for (int y = -1; y <= 1; y++)
        {
            if (x == 0 && y == 0)
                continue; // 跳过中心节点

            int nodeX = node.gridPos.x + x;
            int nodeY = node.gridPos.y + y;

            if (nodeX >= 0 && nodeX < mapX && nodeY >= 0 && nodeY < mapY)
                neighbors.Add(Map[nodeX, nodeY]);
        }
    }
    return neighbors;
}

void RetracePath(Node startNode, Node endNode)
{
    List<Node> path = new List<Node>();
    Node currentNode = endNode;

    while (currentNode != startNode)
    {
        currentNode.spriteRenderer.color = Color.green;
        path.Add(currentNode);
        currentNode = currentNode.parent;
    }
    path.Reverse();
}

注：通过 parent 从终点一路回溯到起点。

④ 扩展邻居节点

对当前节点的邻居：

  • 不可行走或已在 closeList → 跳过
  • 计算 newGCost；若比原 gCost 更优，或该节点还不在 openList，则更新 g/h、设置 parent，并加入 openList

foreach (var node in gridManager.GetNeighbors(currentNode))
{
    if (!node.isWalkable || closeList.Contains(node))
        continue;

    float newGCost = currentNode.gCost + GetDistance(currentNode, node);

    if (newGCost < node.gCost || !openList.Contains(node))
    {
        node.gCost = newGCost;
        node.hCost = GetDistance(node, targetNode);
        node.parent = currentNode;

        if (!openList.Contains(node))
            openList.Add(node);
    }
}

说明

  • 邻居节点初始 gCost 为 0，第一次访问时 newGCost > node.gCost 仍成立，且不在任何队列 → 一定会进入 openList。
  • 赋值后若发现更短路径会更新；往回走时 newGCost 更大，不会错误覆盖。

return diagonalCost * Mathf.Min(xDis, yDis) + straightCost * Mathf.Abs(xDis - yDis);

4. 多单位 A* 的优化

优化 说明
优先队列 + HashSet openList 用 Priority Queue（根为最小 F，O(log n) 取最优点）；closeList 用 HashSet（O(1) 查找是否已探索）
相同起点 直接复用缓存路径
起点不同、终点相同 用 Dijkstra 从终点反向扩散，预计算每个格子的行进方向，多单位共享

优先队列：逻辑上是完全二叉树，实现上常用数组 + 下标公式；根节点始终为当前最小代价节点。

要点速查

概念 要点
A* F = g + h，按 F 升序扩展，h 需可采纳（不高估）
open / close 待探索 / 已探索，避免重复扩展
parent 记录最短路上的前驱，终点回溯得路径
四向 / 八向 曼哈顿 / 直10斜14 代价
多单位 优先队列提速；同终点可反向 Dijkstra 预处理
