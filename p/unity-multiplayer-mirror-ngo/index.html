Unity 多人联网：Mirror 与 Netcode for GameObjects（NGO）

  • 核心模型相同：服务器权威 + 状态同步，客户端发请求、服务器改真相再下发。

选型与共同点

维度 Mirror NGO (Netcode for GameObjects)
来源 社区（原 UNET 分支） Unity 官方包
包名 导入 Mirror 资源 / UPM com.unity.netcode.gameobjects
网络实体 NetworkIdentity NetworkObject
行为脚本基类 NetworkBehaviour NetworkBehaviour
总控 NetworkManager（勿挂在 NetworkIdentity 上） NetworkManager（全场景唯一）
状态同步变量 [SyncVar] NetworkVariable<T>
远程调用 [Command] [ClientRpc] [TargetRpc] [Rpc(SendTo.Server)] 等
Transform 同步 NetworkTransform（Mirror 组件） NetworkTransform（NGO 组件）
传输层 Telepathy / KCP 等（常 TCP+UDP 组合） Unity Transport（UTP，偏 UDP）
公网联机 需自建 / Relay 第三方 Unity Relay + Lobby 官方链路
学习资料 项目内 Mirror 笔记、社区多 官方文档、Multiplayer Tools

共同原则

  1. 每个联网对象有权威端（通常是 Server）。
  2. 客户端只发指令（RPC），玩法判定在服务器。
  3. 持久状态用同步变量（SyncVar / NetworkVariable），持续位置用 NetworkTransform。
  4. 场景中通常只有一个激活的 NetworkManager。

客户端 / 服务器 / Host

Mirror 与 NGO 都不是固定两个 exe，而是通过打包方式 + 启动 API 区分角色。

形态 说明 Mirror NGO
客户端 渲染、输入、发 RPC；不做权威判定 StartClient() StartClient()
专用服务器 无头、无渲染；权威逻辑 StartServer() + Headless Build StartServer() + Dedicated Server Build
Host 同进程兼 Server + Client StartHost() StartHost()
能力 Mirror NGO
是否服务器 isServer IsServer（Host 也为 true）
是否客户端 isClient IsClient（Host 也为 true）
是否 Host isServer && isClient IsHost
是否本地拥有者 isOwned / hasAuthority（版本用语） IsOwner
已入网 netIdentity != null 等 IsSpawned

Host 注意：写逻辑时用 IsServer / isServer 做玩法判定，不要只用 IsClient，否则 Host 会漏逻辑。

连接是如何建立的

通用流程（二者逻辑一致）

Mirror：传输与握手（你的笔记要点）

  1. 传输层：Mirror 通过 Transport 抽象使用 TCP / UDP（默认封装，一般不直接写 Socket）。
    • TCP：连接、鉴权、关键可靠消息。
    • UDP：位置等实时数据（上层可加重传）。
  2. 服务器：StartServer() / StartHost() → 绑定 0.0.0.0:端口 → 监听。
  3. 客户端：StartClient(address, port) → 向目标 IP 发起连接。
  4. TCP 三次握手：SYN → SYN+ACK → ACK（Mirror Transport 自动完成）。
  5. 应用层握手：版本协商、可选身份验证 → 成功则同步房间/玩家列表。
  6. NetworkManager：连接全流程入口；回调如 OnServerConnect、OnClientConnected。
Host vs 专服 区别
Headless Server 独立进程监听；客户端连 IP:Port
Host 无独立服程序；StartHost() 本地同时开服+客；其他客户端连 Host 的局域网 IP

Mirror.NetworkManager 注意

  1. 每场景仅一个激活的 NetworkManager（单例）。
  2. 不要把 NetworkManager 挂在带 NetworkIdentity 的对象上（场景加载时可能被禁用）。
  3. 一个节点树只允许一个 NetworkIdentity，挂在父节点；父链上不能再有 Identity。

NGO：传输与鉴权

项 说明
Unity Transport 默认 UTP（UDP），在 NetworkManager 选 Transport
连接 StartServer / StartHost / StartClient，地址端口在 Transport / NetworkConfig
鉴权 ConnectionApproval = true + ConnectionApprovalCallback
公网 Unity Relay，客户端无需主机公网 IP
断开 NetworkManager.Singleton.Shutdown()

修饰符 / 执行上下文（Mirror 独有语法）

Mirror 含义
[Server] 仅服务器执行；客户端调用会警告
[ServerCallback] 仅服务器执行；客户端调用不警告（常用于 Update 里包一层）
[Client] 仅客户端执行
[ClientCallback] 仅客户端执行；服务器调用不警告

NGO 无上述特性标签，用 if (IsServer) / if (IsClient) / if (IsOwner) 分支代替。

RPC 对照

方向 Mirror NGO (2.x)
客户端 → 服务器 [Command]，方法名建议 Cmd 开头 [Rpc(SendTo.Server)]
服务器 → 所有客户端 [ClientRpc] [Rpc(SendTo.Clients)] 或 SendTo.ClientsAndHost
服务器 → 指定客户端 [TargetRpc] Rpc + RpcParams 指定 ClientId
调用限制 Command 不能从 Server 调 Server RPC 须 Client/Owner 发起；ClientRpc 须 Server 发起

同步变量对照

Mirror [SyncVar] NGO NetworkVariable<T>
谁改 仅 Server 改字段 常用 WritePermission.Server
客户端 只读，自动同步 .Value 读，Server 写
变化回调 [SyncVar(hook = nameof(OnHpChanged))] OnValueChanged += ...
Hook 执行端 在客户端，收到服务器同步后触发 各端订阅，在值变化时触发
UI 更新 在 hook 里改 UI（客户端） 在 OnValueChanged 里改 UI

为什么本地直接改 UI 不会同步给别人？
UI 只是本机表现；要让所有人看到名字/颜色，必须 Command/RPC → 服务器改 SyncVar/NetworkVariable → 自动同步到所有客户端 → hook/OnValueChanged 更新 UI。

public NetworkVariable<FixedString64Bytes> PlayerName = new(
    default,
    NetworkVariableReadPermission.Everyone,
    NetworkVariableWritePermission.Server);

[Rpc(SendTo.Server)]
void SubmitNameRpc(FixedString64Bytes name)
{
    PlayerName.Value = name;
}

public override void OnNetworkSpawn()
{
    PlayerName.OnValueChanged += (_, newVal) => nameText.text = newVal.ToString();
}

场景与生成

主题 Mirror NGO
离线场景 Offline Scene（断线后加载） 自行用 SceneManager + Shutdown 处理
在线场景 Online Scene（连接后加载） NetworkSceneManager 或手动加载
切场景 NetworkManager.singleton.ServerChangeScene(name) NetworkSceneManager.LoadScene
玩家生成 在 Online Scene 生成，不在 Offline Scene Server Spawn Player Prefab
跨场景保留 NetworkManager / 玩家上 DontDestroyOnLoad 同上

Mirror 生成注意：玩家默认在 Online Scene 生成；跨场景不消失需 DontDestroyOnLoad（通常挂在 Manager 或 Player 根上）。

Mirror 项目脚本架构（你的 VR 工程）

脚本 职责
VRNetworkManager 总控：初始化联网/断网场景，Discovery、重连；管理 Host/Client
VRNetworkDiscovery 局域网发现（广播/响应房间）
VRNetworkLobbyUI 大厅入口：监听发现事件，按配置自动发现或手动进房
VRNetworkReconnect 断线重连
VRNetworkPlayer 玩家可见性、XR Rig 挂接
NetworkMessages 自定义消息数据结构（若用 Message 系 API）

对应到 NGO 时：Manager → NetworkManager + 自定义 Lobby；Discovery → Relay/Lobby 或局域网广播自写；Reconnect → 监听 OnClientDisconnectCallback 后重连；Player → NetworkObject + Owner 启用 XR。

常见坑（两边通用）

现象 原因
RPC 不执行 未 Spawn / 调用端不对（Client 调了 ClientRpc）
别人看不到我 未 Spawn、预制体未注册
位置回弹 客户端和服务器同时改 Transform
变量不同步 客户端直接改 SyncVar / NetworkVariable
VR 头身分离 Rig 与 Network Player 根节点混用
现象 Mirror 特有 NGO 特有
Manager 失效 Manager 挂在 NetworkIdentity 上 多个 NetworkManager
玩家不在线场景 未设 Online Scene Prefab 未进 Network Prefabs

调试：Mirror 可多开 Build + Editor；NGO 用 Multiplayer Play Mode + Multiplayer Tools Profiler。
