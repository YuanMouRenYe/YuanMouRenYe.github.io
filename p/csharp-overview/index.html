C# 语言基础与常用语法知识点概要。

C# 知识点概要

1. 方法重载和重写

说明
重载 同一类中多个同名、形参不同的方法
重写 用 virtual / override 使基类与派生类方法行为不同

注：抽象类的抽象方法必须在派生类重写；虚方法可以重写。

2. 面向对象三大特征

封装

将属性与方法结合，限制外部直接访问内部数据；通过公共方法访问，提高安全性。

继承

  • 作用：提高代码复用与可维护性，集中管理子类公共属性。
  • 特性：传递性、单根性（C# 单继承）。

多态

通过统一接口调用不同类对象，得到不同结果；主要体现为重载与重写。

3. 值类型和引用类型

值类型

项 说明
存储 直接存实际数据
类型 int、float、bool、char、double、struct、enum 等
位置 一般在栈（作为局部变量时）

引用类型

项 说明
存储 栈上存引用（地址），堆上存实际数据
类型 string、object、class、interface、delegate
位置 对象数据在堆

注：类的成员变量在堆中；方法内局部变量：值类型与引用各自按规则分配（值在栈，引用变量在栈、对象在堆）。

4. 堆与栈

栈

  • 内容：值类型、引用（地址）、参数、局部变量、返回地址等
  • 生命周期：随作用域自动创建/销毁
  • 内存：地址连续，无碎片问题

堆

  • 内容：引用类型的实际数据（如 new 出的对象）
  • 生命周期：由 GC 回收；有引用指向则继续存在
  • 内存：地址分散，可能有碎片

5. 装箱与拆箱

说明
装箱 值类型 → 引用类型（如 object）
拆箱 引用类型 → 值类型

常见触发

  • 值类型赋给 object 或接口变量
  • 以值类型调用 object/接口方法
  • 非泛型集合（如 ArrayList）存值类型

避免

  • 使用泛型集合 List<T>、泛型方法
  • 避免把值类型转为 object
  • 避免在 Unity 周期函数中频繁装拆箱
  • 高频值类型可用 NativeArray<T>

6. private、public 与 protected

修饰符 访问范围
public 任意类、任意成员
private 仅当前类
protected 当前类 + 派生类

7. ArrayList 与 List

ArrayList List<T>
类型 非泛型，元素为 object 泛型，类型安全
开销 存取可能装/拆箱 无类型转换，更高效

8. 接口与抽象类

接口

  • 完全抽象（只定义不实现，C# 8+ 可有默认实现例外此处按原文）
  • 多继承
  • 成员一般为 public
  • 多类实现同一接口 → 同一调用、不同效果

抽象类

  • 可有抽象方法（不实现）+ 具体方法（已实现）
  • 单继承
  • 成员可为 public / protected / private（private 不能修饰抽象方法）
  • 子类复用具体方法，并必须实现抽象方法
虚方法 virtual 抽象方法 abstract
实现 可有默认实现，子类可选 Override 无实现，子类必须 Override
所在 抽象类或普通类 仅抽象类

9. Sealed 关键字

  • 修饰类：禁止被继承
  • 修饰方法：禁止派生类重写（需与 override 等配合使用场景见语言规范）

10. unsafe 关键字

  • 作用：允许指针操作，直接访问内存
  • 用法：可修饰代码块、方法、类、结构体

bool TryParseInt(string input, out int result)
{
    if (int.TryParse(input, out result))
        return true;
    result = 0;
    return false;
}

string str = "123";
if (TryParseInt(str, out int num))
    Console.WriteLine(num); // 123
else
    Console.WriteLine(num); // 0

注意

  2. 普通参数传的是引用的副本；ref 传的是变量本身的引用。

17. 字典（Dictionary）内部实现原理

1) 概念

基于哈希表的键值对集合。

2) 内部核心结构

结构 作用
buckets[] 桶数组，存 entries 的索引；下标为哈希值；初始为质数长度，扩容为下一质数
entries[] 存条目结构体：键、值、Next（下一索引）
辅助字段 count、version、freeList、freeCount

关系：buckets[哈希值] = entries 中的索引（链表头）。

3) 哈希冲突（拉链法）

  • 冲突：不同键算出相同桶下标
  • 处理：同桶多条目成链表，Next 串联

4) 关键操作

  • 扩容：负载因子超阈值 → 桶数量增加（常为质数）→ 重算哈希、重新挂链
  • 查找：从桶头遍历链表：比哈希码 → 比键 → 返回值或失败

注：条目=键值对；桶存的是 entries 索引；buckets 下标是哈希值；entries 存真实键值对。

18. C# 与 C++ 中的哈希表

1. 底层实现

C# Hashtable

  • 拉链法（Separate Chaining），桶为链表
  • 使用双散列处理冲突（原文表述）
  • 非泛型，object，有装拆箱
  • 已被 Dictionary<TKey,TValue> 取代，旧代码仍可见

C++ std::unordered_map

  • 开放寻址或拉链法（因编译器/实现而异）
  • 模板泛型，编译期定类型
  • 内存通常更紧凑

2. 线程安全

C# Hashtable C++ unordered_map
默认 非线程安全 非线程安全
安全用法 Hashtable.Synchronized（锁，性能低） 用户自加 std::mutex 等

3. 内存管理

  • C#：GC 自动管理
  • C++：插入/删除时构造/析构，需关注对象生命周期

4. 性能对比

特性 C# Hashtable C++ std::unordered_map
查找（平均） O(1) O(1)
内存开销 较高（链表 + 装箱） 较低（布局更紧）
线程安全 内置同步包装（性能低） 需手动实现

public class Person
{
    public string Name;
    public int Age { get; set; }
    public int Add(int a, int b) => a + b;
}

class ReflectionDemo
{
    static void Main()
    {
        Type personType = typeof(Person);

        foreach (var prop in personType.GetProperties())
            Console.WriteLine($"- 属性：{prop.Name}，类型：{prop.PropertyType.Name}");

        foreach (var method in personType.GetMethods())
            if (method.DeclaringType == personType)
                Console.WriteLine($"- 方法：{method.Name}");

        Person person = (Person)Activator.CreateInstance(personType);

        personType.GetField("Name").SetValue(person, "Alice");
        personType.GetProperty("Age").SetValue(person, 25);

        object result = personType.GetMethod("Add").Invoke(person, new object[] { 3, 5 });
        Console.WriteLine($"3 + 5 = {result}"); // 8
    }
}

用途：分析类型、动态创建对象、调用方法、访问成员、加载未知程序集（插件）等。

21. C# 编译过程中的文件与概念

Unity 中 C# 大致生命周期

  1. 编写：C# 源代码（.cs）
  2. 编译：编译为 IL + 元数据，打包为程序集（.dll）
  3. 分发/部署：发布程序集
  4. 执行（IL2CPP）：IL2CPP 将 IL + 元数据转为 C++ → 再编译为本地机器码 → 链接为可执行文件 → 设备上运行本地码
  flowchart LR
    A[.cs 源码] --> B[IL + 元数据 / 程序集]
    B --> C[IL2CPP]
    C --> D[C++]
    D --> E[本地机器码]

22. 小知识点总结

1）A 实例化赋给 B，删除 B，A 是否存在？

存在。B 只是复制了栈上的引用；堆上由 A 创建的对象仍在（若无其它引用则由 GC 回收）。

2）foreach 中删除元素报错怎么办？

foreach 不能在迭代时删元素。应先记录索引或 key，遍历结束后再删除。

3）函数里多次 string += 产生 GC 怎么办？

用 StringBuilder，减少临时字符串对象。

StringBuilder sb = new StringBuilder(100000); // 预估容量，减少扩容

for (int i = 1; i <= 10000; i++)
{

}

4）频繁创建同一类对象如何省内存？

单例模式或对象池。

5）单例模式违反了什么设计原则？

依赖倒置原则（具体类被硬编码依赖，不利于替换与测试）。
