Unity DOTS

1. 基本概念

DOTS（Data-Oriented Technology Stack）是 Unity 基于 ECS 架构 的一套面向数据的技术栈，主要包括：

组成部分 作用
Burst Compiler 将 IL/.NET 字节码编译为高度优化的本地代码
Job System 将计算拆成多任务并行执行
ECS Entity / Component / System 数据驱动架构

2. Burst Compiler

项 说明
优势 将 IL / .NET 字节码 转为高度优化的本地代码，接近 C++ 性能；可利用 CPU 特性做性能关键代码微调
用法 在 Job 结构体上添加 [BurstCompile]

3. Job System

优势

Unity 传统逻辑多在主线程单线程执行；Job System 把计算拆成多个并行任务，充分利用多核 CPU。

[BurstCompile]
struct MathfJob : IJobParallelFor
{
    [ReadOnly] public NativeArray<float> inputData;
    public NativeArray<float> outputData;

    public void Execute(int index)
    {
        outputData[index] = inputData[index] * inputData[index];
    }
}

private NativeArray<float> inputData;
private NativeArray<float> outputData;
private JobHandle jobHandle;

void Start()
{
    // Allocator.Persistent：内存持续到手动 Dispose
    inputData = new NativeArray<float>(1_000_000, Allocator.Persistent);
    outputData = new NativeArray<float>(1_000_000, Allocator.Persistent);
}

void Update()
{
    if (!jobHandle.IsCompleted)
        jobHandle.Complete();

    var job = new MathfJob
    {
        inputData = inputData,
        outputData = outputData
    };
    jobHandle = job.Schedule(inputData.Length, 64);
}

void LateUpdate()
{
    jobHandle.Complete();
}

void OnDestroy()
{
    jobHandle.Complete();
    inputData.Dispose();
    outputData.Dispose();
}

注意事项

  1. 使用 NativeArray<T> 代替普通数组：不受 GC 管理，可跨线程访问（需遵守读写规则）。
  2. 核心 Job 接口：
接口 用途
IJob 一次性计算
IJobParallelFor 批量数据并行处理
IJobFor 复杂并行迭代
IJobParallelForTransform 批量并行处理 Transform
  1. Schedule(总长度, 每批数量) 将 Job 交给线程池；Complete() 阻塞主线程直到 Job 完成（调用后主线程逻辑会等待）。
  2. 在 OnDestroy 中、且 Job 已完成的前提下，必须对 NativeArray 调用 Dispose() 手动释放。

Allocator 简表

类型 说明
Allocator.Temp 帧内临时，最快
Allocator.TempJob 跨帧短生命周期，需在 4 帧内 Dispose
Allocator.Persistent 长期持有，需手动 Dispose

[SerializeField] private Mesh mesh;

private void Start()
{
    EntityManager entityManager = World.DefaultGameObjectInjectionWorld.EntityManager;

    // 创建实体原型（Archetype）
    Entity archetype = entityManager.CreateEntity(
        typeof(Translation),
        typeof(RenderMesh),
        typeof(RenderBounds)
    );

    Entity entity = entityManager.CreateEntity(archetype);

    entityManager.SetSharedComponentData(entity, new RenderMesh
    {
        mesh = mesh,
        material = material,
        layerMask = 1
    });
}

partial class MovementSystem : SystemBase
{
    protected override void OnUpdate()
    {
        float deltaTime = Time.DeltaTime;
        Entities.ForEach((ref Translation position, in PlayerData data) =>
        {
            position.Value.x += deltaTime * data.speed;
        }).ScheduleParallel();
    }
}

原文类名为 MovementSyetem，此处按正确拼写 MovementSystem 整理；Data.speed 应为 data.speed。

ForEach 调度方式

方法 说明
.Run() 主线程同步执行
.Schedule() 异步单线程执行
.ScheduleParallel() 异步多线程并行执行

参数修饰

修饰 含义
ref 可读可写组件
in 只读组件

注：ForEach 用于对符合查询条件的实体执行逻辑（常配合 EntityQuery）；新版本 API 可能使用 SystemAPI.Query 等写法。

DOTS vs 传统 MonoBehaviour

维度 传统 OOP DOTS
数据布局 对象分散在堆 组件数据连续（SOA 友好）
线程 主线程为主 Job + Burst 并行
逻辑 挂在 GameObject 上 System 批量处理实体
GC 较多 NativeContainer 可控分配

要点速查

主题 要点
Burst [BurstCompile] + Job/ECS 数学密集代码
Job NativeArray + Schedule + Complete + Dispose
ECS Entity 是 ID，Component 是数据，System 是逻辑
性能 面向数据、少 GC、多核并行
